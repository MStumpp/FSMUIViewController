FSMUIViewController is an UIViewController extended by Finite State Machine (FSM) functionality. It provides functionality to configure different states and then move between these states.

Configuration of States

This section discusses how to configuration of custom view controller states and fixed view controller states.

Each fsm state definition consists of two different parts:

  • The identifier of the fsm state, which is defined by an arbitrary string, e.g. "state1". There is also a default state.
  • For each fsm state defined before, code to be executed in that state has to be assigned to some view controller state controlled by UIViewController.

The following view controller states are available:

  • tDidInitViewState
  • tDidLoadViewState
  • tWillAppearViewState
  • tDidAppearViewState
  • tWillDisappearViewState
  • tDidDisappearViewState
  • tWillUnloadViewState
  • tDidUnloadViewState

Configure an arbitrary State

[[self configureState:@"state1"] onViewState:tDidAppearViewState do:^{
   ...do something
}];

Configure the default State

[[self configureDefaultState] onViewState:tDidAppearViewState do:^{
   ...do something
}];

Moving between States

Move to a certain State

-(void)toState:(int)state;

Do nothing, if you are already in that state.

Move to the default State

-(void)toDefaultState;

Do nothing, if you are already in that state.

Forced move to a certain State

-(void)toStateForce:(int)state;

Do move, even if you are already in that state.

Forced move to the default State

-(void)toDefaultStateForce;

Do move, even if you are already in that state.
